# 七日書 Landing Page 交接方案

## 交接目標

建立一個可讓 Matters 內部、設計端與後續工程端共同接手的 Landing Page 工作包。此階段聚焦內容策略、資料證據、頁面架構與後續實作任務，不處理完整視覺系統定稿。

## 目前已完成

1. 擷取 Matters 官方帳號資料  
   來源為 `https://matters.town/@freewrite`。目前本 repo 已保存 93 篇文章、11 則動態、185 張圖片，並包含 profile、summary、article index、全文 JSON 與圖片 manifest。

2. 完成 Landing Page 內容藍圖  
   頁面以合作機構、贊助企業、公共文化機構為第一優先受眾。讀者參與與作品閱讀作為第二層導流。

3. 完成 Astro 靜態頁面初版  
   頁面位於 `src/pages/seven-day-book.astro`，內容資料位於 `src/data/seven-day-book.ts`。內容已拆成 metrics、themes、caseStudy、tagLinks、publicValues、cta 等資料區塊，方便後續替換與設計套版。

4. 移除舊部署設定  
   原始 Astro 專案帶有舊 GitHub Pages workflow 與 `pro.mashbean.net` CNAME，已在交接 repo 移除，避免誤部署。

## 核心內容方向

首屏需要讓合作機構在 5 秒內理解三件事。

- 七日書是 Matters 發起的連續七日自由書寫計畫。
- 這個活動已累積可被衡量的參與規模與作品成果。
- 它能成為機構或企業投入公共文化議題的合作載體。

主要 CTA 為「洽談合作」。副 CTA 可為「閱讀七日書作品」或「加入下一期七日書」，但不能搶過合作導向。

## 頁面資訊架構

1. Hero  
   標題方向為「七日書，把人生寫成一座公共記憶庫」。主 CTA 指向合作洽談，副 CTA 指向七日書標籤。

2. 數據成果  
   使用 19 個月、133 道題目、3000 多位文友、近 9000 篇故事、八百萬字，並帶來源與日期。

3. 活動如何運作  
   用七道題目、七天書寫、作品進入 Matters 標籤與活動頁三步說明。補充每月節奏、作家講座、完結報告、大滿貫徽章與鏈上參與憑證。

4. 人生議題地圖  
   分成自我與生命階段、家庭與親密關係、地方與遷徙、物件與感官記憶、數位與當代生活五組。

5. 公共意義  
   主打個人表達、社群連結、公共記憶。可引用核心詞彙，我們、開始、時間、喜歡、關係、選擇、生活、感覺、人生。

6. 合作與贊助案例  
   首推兩廳院藝術出走「給自己的情書」，呈現合作形式、參與規模、講座作家與超高齡社會議題。

7. 作者與作品導流  
   使用標籤入口，不做單一作者排行榜。

8. CTA 收束  
   文案方向為「一起設計下一個能被持續書寫的公共議題」。表單欄位包含合作單位、聯絡人、合作方向、預計時程、補充說明。

## 後續工作建議

1. 在乾淨環境重新安裝依賴並跑檢查  
   先執行 `npm install`、`npm run check`、`npm run build`。若 Astro CLI 還有環境問題，先處理 Node 版本與依賴安裝。

2. 進行內容校稿  
   核對每個數字是否能回到 `research/freewrite` 或 Matters 官方文章。不要擴寫成未被資料支持的成效。

3. 套入 Matters 視覺系統  
   當設計系統完成後，將目前頁面 CSS 改寫成正式 token、元件與 spacing 規範。現階段頁面已有結構，但不是最終視覺。

4. 補上真正合作表單  
   目前 CTA 使用 mailto。後續可改成 CRM 表單、Google Form、Typeform、HubSpot 或內部接案流程。

5. 做瀏覽器驗證  
   必須檢查 desktop、tablet、mobile。重點是首屏訊息、CTA 層級、圖片載入、長文字換行、連結可點擊與資料來源註記。

6. 決定部署策略  
   目前沒有保留 GitHub Pages workflow。後續可依 Matters 既有前端部署流程設定 Vercel、Cloudflare Pages、GitHub Pages 或併回主站。

## 風險與注意事項

- 不能把 3009 參與人次寫成不重複用戶數。
- 不能把拍手、贊助、追蹤者寫成平台總使用者數。
- 不要把合作案例擴寫成長期合作關係，除非 Matters 內部有額外證據。
- 圖片素材來自官方帳號擷取，正式上線前仍需確認授權、肖像與合作單位露出規範。
- 讀者導流不應壓過合作洽談 CTA。
